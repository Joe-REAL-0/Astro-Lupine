# 🐺 The Astro-Lupine (星际狼客)

[English](./README.md) | [中文](./README_zh.md)

![Version](https://img.shields.io/badge/version-v1.0.0-blue)
> 一款为 **《杀戮尖塔 2》(Slay the Spire 2)** 打造的自定义角色模组，基于 Godot + C# 以及 [BaseLib](https://github.com/Alchyr/BaseLib-StS2) 模组框架开发。

**星际狼客 (The Astro-Lupine)** 是《杀戮尖塔 2》的一名全新角色。他是一只游走于星海与尖塔之间的灰狼兽人。采用赛博朋克与废土风格的美术设计，他将通过精准的 **寄存器运算系统** 粉碎敌人。

---

## ✨ 模组特色

- 🔧 **寄存器系统 (Registers)** — 全新的角色专属常驻 Buff 机制，包含攻击、防御、抽牌三个核心寄存器。
- 📖 **读取 (Read)** — 将寄存器的数值附加到卡牌的基础数值上，且不受虚弱/脆弱等常规 Debuff 影响。
- ✏️ **写入 (Write)** — 强制将卡牌数值覆盖赋值给寄存器，实现无尽的数值膨胀。
- 🎴 **75+ 张卡牌** — 包含初始卡牌和丰富的卡池，可构筑出多种流派。
- 🌐 **多语言支持** — 完整支持 中文 / 英文 本地化。

## 🎮 核心机制

### 寄存器

战斗开始时，角色将自动获得三个寄存器 Buff：

| 寄存器 | 初始值 | 作用 |
| -------- | :----: | -------- |
| 攻击寄存器 ⚔️ | 6 | 控制伤害输出成长 |
| 防御寄存器 🛡️ | 5 | 控制生存能力下限 |
| 抽牌寄存器 🃏 | 2 | 控制运转引擎效率 |

> 所有寄存器的最小值均为 **1**，永远不会降到 0。

### 关键字

| 关键字 | 计算公式 | 说明 |
| ------- | ------------------- | ----------- |
| **读取 (Read)** | `最终数值 = 卡牌基础值 + 对应寄存器层数` | 在所有常规 Buff/Debuff 计算完毕后**最后应用**。 |
| **写入 (Write)** | `寄存器层数 := 指定数值` | 用指定数值直接覆盖寄存器的现有数值，而非加减运算。 |
| **增量/减量 (Inc/Dec)** | `寄存器层数 += 变化值` | 基于当前数值进行相对加减。 |

### 初始卡组 (12 张)

| 卡牌 | 数量 | 费用 | 效果简述 |
| ---- | -- | :--: | -------------- |
| 打击 (Strike) | 5 | 1 | 0 基础伤害 + **读取** |
| 防御 (Defend) | 5 | 1 | 0 基础格挡 + **读取** |
| RA++ | 1 | 1 | 攻击寄存器 **+1** (增量) |
| RD++ | 1 | 1 | 防御寄存器 **+1** (增量) |

## 🚀 编译与安装

### 前置要求

- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0)
- [Godot 4.5+](https://godotengine.org/) (.NET / C# 版本)
- 已安装《杀戮尖塔 2》游戏本体及 [BaseLib](https://github.com/Alchyr/BaseLib-StS2) 框架

### 第一步：编译 DLL

```bash
cd "Astro Lupine"
dotnet build -c Release
```

编译输出文件将位于 `bin/Release/net9.0/AstroLupine.dll`。

### 第二步：导出 PCK

1. 使用 Godot 4.5+ 打开项目
2. **项目 (Project) → 导出 (Export) → 添加 Windows Desktop 预设**
3. 点击 **导出 PCK/ZIP (Export PCK/ZIP)** 并保存为 `AstroLupine.pck`

### 第三步：部署到游戏

将以下文件复制到游戏的 Mods 目录中：

```text
<STS2 安装目录>/Mods/AstroLupine/
├── AstroLupine.dll
├── AstroLupine.pck
└── AstroLupine.json
```

启动游戏，您即可在角色选择界面看到这位全新角色。
