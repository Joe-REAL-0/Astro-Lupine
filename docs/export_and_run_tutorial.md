# 杀戮尖塔2 (Slay the Spire 2) 模组打包与运行教程

## 核心问题解答

**Q: 暂时不添加任何图片素材，就这么打包并导出项目，可以正常在游戏中运行吗？**

**A: 完全可以。**
在 Godot 引擎和《杀戮尖塔2》的 BaseLib 框架下，缺失的贴图、特效等素材会自动回退（Fallback）为引擎内置的占位符（例如透明的方块、默认的 Godot 图标或者干脆不显示），**这不会导致游戏或代码逻辑崩溃**。你可以借此机会在没有任何美术资源的干扰下，纯粹地测试你的卡牌数值、寄存器逻辑和连招机制。

---

## 模组打包与导入教程

因为《杀戮尖塔2》使用 Godot 引擎和 C# 开发，标准的模组（Mod）往往由两个核心文件组成：
1. **DLL 文件**（包含所有 C# 代码逻辑）
2. **PCK 文件**（包含所有 Godot 引擎需要的图片、音频、场景等资源文件）

即使你现在没有图片，也建议按照标准流程进行操作。以下是完整的打包和部署步骤：

### 第一步：编译 C# 代码 (生成 DLL)
1. 在你的项目根目录（`c:\Users\Joe\Documents\Astro Lupine\`）下打开终端 / PowerShell。
2. 运行命令：
   ```bash
   dotnet build -c Release
   ```
   *(注：在开发测试阶段，使用默认的 `dotnet build` 生成 Debug 版本也可)*
3. 编译成功后，前往路径：`bin\Release\net9.0\`（或者 `bin\Debug\net9.0\`）。
4. 找到并复制 **`AstroLupine.dll`** 文件。

### 第二步：导出游戏资源 (生成 PCK)
1. 打开 **Godot 4.3+** 引擎，载入当前项目 `Astro Lupine`。
2. 点击顶部菜单栏的 **项目 (Project)** -> **导出 (Export)**。
3. 点击顶部的 **添加 (Add...)**，选择一个平台（例如 **Windows Desktop**）。
   *(注：如果你看到报错提示没有导出模板，请根据 Godot 提示点击“管理导出模板”并下载安装)*
4. 在导出窗口底部，**不要**点击“导出项目 (Export Project)”，而是点击旁边的 **导出 PCK/ZIP (Export PCK/ZIP...)**。
5. 将生成的文件命名为 **`AstroLupine.pck`** 并保存。

### 第三步：安装与运行
1. 找到你电脑中《杀戮尖塔2》的安装目录，通常在：
   `C:\Program Files (x86)\Steam\steamapps\common\Slay the Spire 2\`
2. 在游戏根目录下，找到（或手动创建一个）名为 **`Mods`** 的文件夹。
3. 在 `Mods` 文件夹内，为你自己的模组创建一个专属文件夹，命名为 **`AstroLupine`**。
   完整路径看起来像这样：`...\Slay the Spire 2\Mods\AstroLupine\`
4. 将你刚才生成的 **`AstroLupine.dll`** 和 **`AstroLupine.pck`** 两个文件粘贴到这个文件夹中。
5. （可选）如果你编写了 `mod.json` 或 `metadata.json` 用于向 BaseLib 声明 Mod 信息，也请一并放进去。

### 第四步：进游戏测试
1. 通过 Steam 启动《杀戮尖塔2》。
2. 如果 BaseLib 正常安装且激活，它会在游戏启动时自动扫描 `Mods/` 目录并加载你的 DLL 和 PCK。
3. 在角色选择界面，你应该能看到没有任何封面（或带有默认 Godot 图标）的“星际‘狼’客”角色。
4. 开启一局游戏，你的初始牌库应该恰好是 12 张没有卡图的卡牌，打出后寄存器数字应当按照代码逻辑正常跳动！
